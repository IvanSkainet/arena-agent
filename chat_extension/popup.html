Advanced / experimental

Turn OFF to show a toolbar on every candidate host, even when multiple render the same tool block (Claude call_id 1..N, Mistral echo, Gemini Thought Process + main answer). More visible feedback but sometimes puts a toolbar on top of the composer. When ON, the extension prefers the copy with the highest numeric call_id (v0.14.16).

OFF by default. When ON, the extension will try to mount a toolbar on ANY site (Ollama-webui, LibreChat, ChatUX, etc.) provided the page has a discoverable composer AND the tool block sits inside a chat-shaped ancestor (role=article/log, class contains message/chat/conversation/bubble). Documentation pages don't have both markers so README code fences quoting MCP JSONL are safe. If you see unexpected toolbars on non-chat pages, turn this OFF.
