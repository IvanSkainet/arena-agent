How to enable SHA-256 verified installs (add GITHUB_TOKEN)

Without a token, updates use the anonymous /releases/latest redirect path — GitHub does not publish SHA-256 digests there, so the Install button stays disabled to keep verification mandatory. To enable verified installs:

  1. Create a personal access token at github.com/settings/tokens — fine-grained token with Contents: read-only on the arena-agent repo is enough (no other scopes).
  2. Add it to the bridge's environment. Systemd example: 
    systemctl --user edit arena-bridge
# add:
[Service]
Environment="GITHUB_TOKEN=github_pat_..."
# save, then:
systemctl --user restart arena-bridge
  3. Windows service (nssm) example: use nssm's Environment tab to set GITHUB_TOKEN=... and restart the service.
  4. Docker/Podman example: pass -e GITHUB_TOKEN=github_pat_... at run time.

GH_TOKEN (from gh auth login) is also accepted as a fallback.

How to use an agent token

Replace your master token with the agent token in any request:

curl -H "Authorization: Bearer agent-XXXXXXXX-YYYYYYYYYYYYYYYY" \
     https://your-bridge.example/v1/mobile/devices

The agent token works on every endpoint EXCEPT /v1/agents/* (management still requires master).
