  ADBKeyboard helper (unicode text input)
  

    Non-ASCII payloads (cyrillic, emoji) crash Android's built-in
    input text with NullPointerException. Installing the
    ADBKeyboard helper APK adds an IME that accepts base64-encoded
    UTF-8 broadcasts, and type_text auto-routes non-ASCII
    through it. You'll need to accept an on-device "Install this app?"
    dialog on the phone during install, and then activate ADBKeyboard
    as the current keyboard.
  

  

    
    
    
  

  







  Install any APK (with SHA-256 consent)
  

    Upload the APK to /tmp/arena-apk-staging/ on the
    bridge host first (arbitrary paths are refused for safety). Then
    Prepare to see SHA-256 + signature check + consent token, and
    Install to push it to the phone. HyperOS/MIUI shows an on-device
    "Install this app?" dialog you must accept.
  

  

    
    
    
    
  

  
  
  







  Camera (take a photo, list DCIM, pull to bridge)
  

    Launches the phone's default camera app via
    android.media.action.STILL_IMAGE_CAMERA, waits for
    the preview, taps the shutter button (auto-detected or from
    shutter_x/y), then polls DCIM for the new photo and
    pulls it back to the bridge (optionally downscaled). Some Android
    versions block programmatic shutter — if auto-detect fails, dump
    the UI and pass explicit shutter coordinates.
