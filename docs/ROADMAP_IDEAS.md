# База Идей для развития Arena Agent (V2 Roadmap)
Проанализирован опыт передовых CLI-инструментов и ИИ-агентов (MCP-SuperAssistant, Claude Code, Gemini CLI, Codex, OpenClaw, Hermes Agent). 

Вот лучшие функции, которые стоит внедрить в наш проект для достижения максимальной эффективности и удобства.

---

## 1. Контекстно-Зависимые Инструменты (Context-Aware Adapters)
**Откуда взято:** MCP-SuperAssistant (`SiteAdapters`, `ToolDetectors`).
**Как внедрить:**
Вместо того чтобы загружать в агента сразу все 100+ скриптов и MCP-серверов, `agentctl` должен проверять окружение (CWD) перед запуском.
- Если мы в папке с `.git` -> автоматически загружать набор `git_tools`.
- Если находим `package.json` или `pytest.ini` -> загружать специфичные команды для запуска тестов или сборки.
- **Польза:** Экономит контекстное окно ИИ-модели, давая ей только те функции, которые применимы здесь и сейчас.

## 2. Бесшовная интеграция "TDD Loop" (Test-Driven Development)
**Откуда взято:** Claude Code (Cowork), AutoGPT.
**Как внедрить:**
Реализовать команду `agentctl task tdd "описание фичи"`. Агент сам:
1. Пишет тесты.
2. Запускает их (они ожидаемо падают).
3. Пишет код, пока тесты не станут зелеными.
4. В случае зацикливания делает шаг назад (`git reset --hard`) и пробует другой подход.
- **Польза:** Невероятно повышает надежность авто-сгенерированного кода.

## 3. Токен-Менеджмент и Отслеживание Бюджета
**Откуда взято:** Hermes Agent, Claude Code.
**Как внедрить:**
Сейчас `ai_client.py` и `fallback_client.py` просто шлют команды. Нужно добавить встроенный `Cost Tracker` (SQLite база).
Агент сможет сам сказать: *"Выполнение этой задачи стоило 0.05$ (40k токенов)"*. В Дашборд можно будет добавить график потребления.
- **Польза:** Защита от непредвиденных трат, если Агент уйдет в бесконечный цикл.

## 4. Защитный режим (Dry Run / Permission Scopes)
**Откуда взято:** MCP-SuperAssistant (менеджмент доступов).
**Как внедрить:**
Разделить команды на "Безопасные" (чтение, статус) и "Опасные" (rm, git push, kill, db_drop).
Для "Опасных" команд мост (Bridge) должен запрашивать подтверждение (Y/n), которое может выводиться прямо в Дашборде как Push-уведомление (через SSE), ожидая клика пользователя.
- **Польза:** Можно смело оставлять агента работать в фоне на продакшене.

## 5. Семантический поиск по проекту (Local RAG)
**Откуда взято:** OpenClaw, OpenCode.
**Как внедрить:**
Обычного `grep` часто не хватает, чтобы найти нужный класс. Нужна команда `agentctl mcp tools semantic-search`. Она будет использовать легковесную векторную БД (например, ChromaDB или просто TF-IDF на Python без тяжелых зависимостей) для индексации рабочего репозитория.
- **Польза:** ИИ сможет мгновенно ориентироваться в огромных кодовых базах.

## 6. Snapshot & Rollback (Машина Времени)
**Откуда взято:** Claude Code.
**Как внедрить:**
Перед каждой сложной командой Агент автоматически выполняет `agentctl proj snapshot`. Это архивирует текущее состояние директории (даже файлы, не добавленные в Git) во временный кэш. В случае фатальной ошибки можно сказать "Верни как было 5 минут назад".
- **Польза:** Нулевой страх сломать проект.

## 7. Генератор Shell-команд с авто-объяснением
**Откуда взято:** Gemini CLI / GitHub Copilot CLI.
**Как внедрить:**
Иногда пользователю не нужен ИИ, исполняющий код. Ему нужна сложная bash-команда. Мы можем добавить `agentctl ask "как найти все файлы больше 1ГБ и удалить их"`. Мост сгенерирует команду, объяснит флаги, и выведет в консоль с предложением нажать Enter для выполнения.

---
### Резюме для внедрения (Что делаем в первую очередь?):
1. **Snapshots (Машина Времени)** — реализуется легко (zip/tar), пользы максимум.
2. **Context-Aware Adapters** — позволит агенту быть умнее "из коробки".
3. **Безопасный режим с подтверждением через Дашборд (SSE)** — красиво и функционально.
