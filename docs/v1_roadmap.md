# Arena Agent v1.0 Roadmap

## Общая Философия
Агент должен быть полностью локальным, автономным, кроссплатформенным (Linux/Windows/macOS), и не зависящим от конкретной AI-песочницы (Arena, GLM, ChatGPT).

## 1. Универсальный AI Bootstrap (AI-Agnostic)
Текущие скрипты (`b.sh`, `upload.sh`) написаны на Bash и используют `jq` и `curl --curves X25519`. Это может не сработать в AI-окружениях без Bash (например, чистый Python Code Interpreter).
**Решение:** Написать чистый `bootstrap.py`, использующий стандартную библиотеку `urllib.request`. Python есть везде, где есть ИИ-интерпретатор.

## 2. Pythonic CLI (agentctl.py)
Переписать `agentctl` (сейчас это bash-скрипт) на чистый Python. Это сделает платформу на 100% совместимой с Windows.

## 3. Real-Time Streaming (SSE)
Сейчас `b.sh` отправляет запрос и ждет его завершения. Это плохо для долгих процессов.
**Решение:** Настроить мост (Bridge) на потоковую передачу вывода через Server-Sent Events (SSE).

## 4. Git-Backed State & Auto-Rollback
Директория `~/arena-agent` должна быть Git-репозиторием. Каждое действие сопровождается коммитом. При ошибках можно легко сделать `git revert`.

## 5. Visual Dashboard
Локальный Web-интерфейс на `http://127.0.0.1:8765/gui`, где пользователь может смотреть логи и управлять процессами.
