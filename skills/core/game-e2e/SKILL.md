# Игровое E2E-Тестирование (Game E2E Simulation & Saving)

Этот навык обучает Агента проводить автоматические E2E-тесты консольных игр, РПГ, новелл или графических приложений на Windows, используя сохраненные состояния (сейвы).

---

## 🛠️ АЛГОРИТМ ТЕСТИРОВАНИЯ ИГРЫ:

1. **Подготовка Сейв-Стейтов (Save States):**
   * Перед тестом ИИ должен найти, где лежат сейвы игры (например, в директории `data/saves/` или в профиле пользователя).
   * ИИ может подготовить `N` сейвов для проверки `N` игровых ситуаций (например: `save_boss_fight.dat`, `save_inventory_full.dat`).
   * Перед началом теста скопируй нужный сейв-стейт в рабочий сейв игры:
     ```cmd
     copy /y data\\saves\\save_boss_fight.dat data\\active_save.dat
     ```

2. **Запуск Игрового Процесса:**
   * Запусти игру в фоновом/свернутом режиме или в отдельном процессе CMD через `agentctl exec`:
     ```cmd
     agentctl exec "start /b mygame.exe --test-mode"
     ```

3. **Снятие Фоновых Скриншотов / Анализ консоли:**
   * Чтобы "видеть" состояние игры, ИИ делает скриншоты экрана:
     ```cmd
     agentctl desktop shot
     ```
   * Если это консольная игра, перенаправляй вывод консоли в текстовый файл и читай его!

4. **Симуляция Игровых Нажатий (Скормить Ввод):**
   * Используй Desktop namespace для эмуляции нажатий кнопок:
     ```cmd
     agentctl desktop type "loadsave 1"
     agentctl desktop key "enter"
     agentctl desktop key "space"
     ```
   * Ты можешь сымитировать живое прохождение или прожимание диалогов, проверяя, не крашнулся ли процесс.

5. **Проверка результатов и Фиксация Багов:**
   * После прожимания кнопок сделай финальный скриншот или прочитай лог-файл игры.
   * Если лог содержит ключевые слова `crash`, `error`, `exception`, `nullpointer` — зафиксируй ошибку в `reports/tests-report.md`.
   * Воспроизведи звуковой сигнал об ошибке в случае краша:
     ```cmd
     agentctl sys beep --type error
     ```
   * Или сыграй победную мелодию при успешном зеленом тесте:
     ```cmd
     agentctl sys beep --type melody
     ```
