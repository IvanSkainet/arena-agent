# Skill: planning-before-action

Use when a task is non-trivial or changes persistent state.

Checklist:

1. Restate goal and constraints.
2. Inspect current state before changing it.
3. Prefer small reversible steps.
4. Define verification before implementation.
5. If a change is serious: backup first, implement, test, update recovery prompt.
6. Stop and ask if destructive/admin/secret-sensitive action is required.
